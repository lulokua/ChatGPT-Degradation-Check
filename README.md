# ChatGPT 降智检测 Pro

> 一款融合 IP 干净度与 PoW 难度信号的 ChatGPT 用户脚本，用来判断当前会话是否存在“降智风险”。

## 项目简介

很多人判断 ChatGPT 是否“降智”，只看一次 IP 风控分，或者只看一次 PoW difficulty。  
这两种方法都不够稳定。

这个项目的思路是：

- 用 `IPPure` 提供当前 IP 的基础风险分
- 用 `ChatGPT chat-requirements` 返回的 `PoW difficulty` 作为实时侧证
- 用本地历史记录做趋势、波动、持续性分析
- 最后输出一个更接近真实使用体验的综合风险判断

换句话说，它不是简单读取一个分数，而是在做一套面向实际使用场景的“降智风险研判”。

## 核心亮点

| 能力 | 说明 |
| --- | --- |
| 真实出口 IP 检测 | 通过 `chatgpt.com/cdn-cgi/trace` 获取浏览器访问 ChatGPT 时的真实出口 IP |
| IP 干净度分析 | 查询 `IPPure` 风控数据，获取当前 IP 的基础风险评分 |
| PoW 难度采集 | 监听 `chat-requirements` / `sentinel` 响应，提取 `proofofwork.difficulty` |
| 历史趋势算法 | 对本地历史进行会话聚合、时间衰减、EMA 趋势、波动模式识别 |
| 融合判定 | 将 `IPPure + 历史行为 + PoW` 合并成统一的 `compositeRisk` |
| 风险可视化 | 提供迷你卡片、全屏详情页、风险指标、趋势提示 |
| 本地存储 | 所有历史样本保存在浏览器本地，不依赖外部数据库 |
| 附加统计 | 内置模型使用统计，方便查看当天模型切换情况 |

## 它是怎么判断“有没有降智”的

这套脚本不依赖单一信号，而是同时看三类证据：

### 1. 当前 IP 质量

- 当前 `IPPure` 风控分是多少
- 这个 IP 是不是长期稳定
- 最近有没有频繁换节点

### 2. 历史行为轨迹

- 最近几次检测是在变好还是变差
- 分数是否忽高忽低，像共享/轮转节点
- 是否出现连续恶化、连续高风险

### 3. ChatGPT PoW 难度

- 当前 `PoW difficulty` 是高还是低
- 最近一段时间 PoW 是稳定偏高还是持续偏低
- PoW 信号与 IP 风险结论是否一致

最后脚本会输出：

- `compositeRisk`：综合风险分
- `stability`：稳定性
- `confidence`：置信度
- `volatility`：波动指数
- `trend`：趋势方向
- `verdict`：最终结论

## 为什么这版比“只看一个分数”更可靠

### 只看 IP 分数的问题

- 单次分数会被短期波动影响
- 无法区分“当前好”还是“最近一直在恶化”
- 无法识别共享/轮转节点的锯齿波动

### 只看 PoW 的问题

- PoW 会受到问题复杂度、页面状态、请求路径影响
- 单次 PoW 低，不等于一定降智
- PoW 更适合做辅助证据，不适合单独判死刑

### 这套方案的优势

- IP 风险给基础盘
- 历史算法看趋势和稳定性
- PoW 给当前入口质量提供补充证据
- 多信号一致时提高置信度，冲突时降低置信度

## 安装方式

### 环境要求

- 浏览器：Chrome / Edge / Firefox
- 用户脚本管理器：`Tampermonkey` 或 `Violentmonkey`

### 安装步骤

1. 安装用户脚本管理器
2. 将 [ChatGPT降智检测.user.js](./ChatGPT降智检测.user.js) 导入脚本管理器
3. 打开 `https://chatgpt.com/`
4. 首次进入页面后，脚本会自动注入侧边检测面板

## 使用方法

1. 打开 ChatGPT 网页
2. 等脚本自动完成当前出口 IP 检测
3. 发送几条真实消息，让脚本积累样本
4. 点击侧边栏按钮查看迷你卡片
5. 点击“查看详细分析”进入完整分析页

建议：

- 不要只看第一次结果
- 至少积累几次真实会话，再看综合结论
- 如果你经常换节点，先固定一个节点再观察

## 结果怎么看

### 风险等级

| 综合结果 | 含义 | 建议 |
| --- | --- | --- |
| `0-25` | 风险低，通常较稳定 | 可以继续使用 |
| `26-40` | 灰区，需要结合趋势和稳定性看 | 建议持续观察 |
| `41-55` | 风险偏高，已经不够稳 | 建议准备换节点 |
| `56+` | 风险很高，明显不适合继续用 | 建议直接更换 |

### 几个最重要的指标

| 指标 | 含义 |
| --- | --- |
| `compositeRisk` | 综合风险，最终核心评分 |
| `stability` | 历史是否平稳，越高越好 |
| `confidence` | 当前结论是否可靠，越高越可信 |
| `volatility` | 波动大小，越高越不稳定 |
| `trend` | 最近趋势是在恶化、改善还是持平 |
| `powRisk` | PoW 维度风险，越高说明入口质量越弱 |

## 适合哪些人

- 经常切换代理 / 节点的 ChatGPT 用户
- 想判断某个 IP 是否容易“降智”的用户
- 想把 PoW 和 IP 干净度结合起来看的人
- 想做更长期、可积累的节点质量观察的人

## 隐私与数据说明

本项目默认只在本地浏览器存储数据，不会建立远程用户数据库。

本地会保存的主要内容：

- IP 风控历史分数
- 时间戳
- 出口 IP
- PoW difficulty 历史
- 模型使用统计

说明：

- 这些数据保存在浏览器 `localStorage`
- 脚本不会把你的聊天内容上传到本项目服务器
- 为了识别模型和 PoW，脚本会在页面本地解析部分请求/响应，但结果仅用于本地计算

## 适用边界

请把它理解为“风险评估工具”，而不是“绝对真相机器”。

它能显著提高判断质量，但仍然有边界：

- 单次 PoW 不能直接代表真实体验
- 样本太少时，结论会偏保守
- 如果 `IPPure` 回退到非指定 IP 查询，当前评分可能和真实出口存在偏差
- ChatGPT 的网页结构、接口字段未来如果变化，脚本可能需要跟进更新
